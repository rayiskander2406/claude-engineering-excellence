# Documentation Generation

Generate comprehensive documentation for the specified code.

## Documentation Types

### API Documentation
- Function/method signatures
- Parameter descriptions
- Return value descriptions
- Exception/error documentation
- Usage examples

### README Sections
- Overview/description
- Installation instructions
- Quick start guide
- Configuration options
- API reference
- Contributing guidelines

### Architecture Documentation
- System overview diagram (Mermaid)
- Component descriptions
- Data flow
- Key design decisions
- Dependencies

### Inline Documentation
- JSDoc/Javadoc/docstring comments
- Complex algorithm explanations
- Non-obvious code explanations

## Documentation Standards

### Good Documentation
- Explains WHY, not just WHAT
- Includes examples
- Keeps examples up to date
- Uses consistent terminology
- Is scannable (headers, lists, code blocks)

### Format
- Use Markdown
- Use Mermaid for diagrams
- Include code examples
- Link to related documentation

## Output Format

Generate documentation that:
1. Matches the project's existing documentation style
2. Is immediately usable (copy-paste ready)
3. Includes all public API elements
4. Has practical examples
5. Uses proper formatting

## Target

$ARGUMENTS

Specify what to document:
- A file or module
- A class or function
- An API endpoint
- A feature or subsystem
