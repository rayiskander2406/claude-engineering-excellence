# Code Explanation

Analyze and explain the specified code in detail.

## Explanation Levels

### High-Level Overview
- What does this code do? (one sentence)
- Why does it exist? (business purpose)
- How does it fit into the larger system?

### Detailed Walkthrough
- Step-by-step execution flow
- Key data transformations
- Important decision points
- Error handling approach

### Technical Deep Dive
- Algorithm complexity (Big O)
- Design patterns used
- Trade-offs made
- Potential edge cases

## Questions to Answer

1. **What**: What is this code's purpose?
2. **How**: How does it accomplish its purpose?
3. **Why**: Why was it designed this way?
4. **When**: When is this code executed?
5. **Dependencies**: What does it depend on? What depends on it?

## Explanation Format

### Summary
One paragraph overview suitable for someone unfamiliar with the code.

### Flow Diagram
Mermaid diagram showing the execution flow or data flow.

### Key Concepts
- Important variables and their purposes
- Critical functions and what they do
- Non-obvious logic explained

### Gotchas
- Things that might surprise you
- Common mistakes when modifying
- Non-obvious side effects

### How to Modify
- Where to make common changes
- What tests to run
- What could break
